Quick Start

Getting started with the project is really simple. Make sure to run yarn or npm install to install all required dependencies. Next up, run the appropriate command, depending on your needs:

yarn start or npm start

Runs the app in development mode. Open http://localhost:3000 to view it in the browser.

The page will automatically reload if you make changes to the code. You will see the build errors and lint warnings in the console.

yarn build or npm build

Builds the app for production to the build folder. It correctly bundles React in production mode and optimizes the build for the best performance.

The build is minified and the filenames include the hashes.

Your app is ready to be deployed.

yarn test or npm test

Runs the test watcher in an interactive mode. By default, runs tests related to files changed since the last commit.

Architecture

Routing

The main application entrypoint is located inside src/App.js and it consists mainly of the Router instance that iterates over the available routes. The available routes are also declared inside src/routes.js.

In order to implement the dynamic layouts required by Shards Dashboard Pro React, a dynamic component layout is used throughout the route definitions. A route definition looks like this:

// src/routes.js

{
  path: "/analytics",
  layout: DefaultLayout,    // The layout style.
  component: Analytics
}

Each layout component is defined inside ./src/layouts. Currently there are only 3 layout components available:

  • Default → src/layouts/Default.js
  • Header Navigation → src/layouts/HeaderNavigation.js
  • Icon Sidebar → src/layouts/IconSidebar.js

Changing a template’s layout would require you to change the appropriate layout property inside a route definition.

State Management

Internally the application is using Flux to manage its state. This can be easily changed in case a more robust solution such as Redux is required.

All the Flux specific elements is available inside the src/flux directory.

The only components using the Flux architecture are the MainSidebar [src/components/layout/MainSidebar/MainSidebar.js] and also the HeaderNavbar [src/components/layout/HeaderNavbar/HeaderNavbar.js] in order to handle the responsive toggle switch and also maintain the state of the active sidebar dropdown items.

Static Data

Throughout the project you will notice certain static data used for demo purposes. All this data is fetched through plain JavaScript files which are located inside the src/data directory.

Styles

All the styles specific to Shards Dashboard Pro are located inside the src/styles directory and consist in the original SCSS files.

The extra components that come with Shards Dashboard Pro React that required custom adjustments are styles separately inside the src/assets directory.
